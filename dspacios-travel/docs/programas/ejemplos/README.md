# Ejemplos de Programas (terceros) — para definir el modelo

Los **programas** son producto de **terceros** y cada proveedor manda su info con una
**estructura distinta**. Antes de decidir cómo modelarlos en el sistema, subimos aquí
ejemplos reales para verlos y definir el manejo más fácil.

## Qué subir y dónde

- **`01-original-proveedor/`** → el archivo **tal cual lo envía el proveedor**
  (PDF / Excel / Word / imagen, como llegue). Idealmente nombra el archivo con el
  proveedor, ej. `transturismo-brasil-2025.pdf`.
- **`02-publicado-dspacios/`** → tu versión **transformada/publicada** del mismo
  programa (como lo armas hoy para vender).

## Lo más útil para decidir bien

Sube **3–5 programas BIEN DISTINTOS entre sí** (los que más varían), con su par
original ↔ publicado cuando se pueda. Con eso identifico:

- Qué datos son **comunes a todos** (nombre, días/noches, vigencia, "desde", incluye/no incluye, itinerario, hoteles, precios por categoría…).
- Qué partes son **libres/variables** por proveedor.
- Si conviene: (a) plantilla flexible por secciones, (b) adjuntar el PDF del proveedor + pocos campos clave para la vitrina, o (c) un híbrido.

> Tip: si los archivos son pesados, también puedes pasarme solo 1–2 páginas
> representativas de cada uno.
